devops实践中的python库

说明
-----
运维脚本写的多。工程方面有些不足。因此用这个项目，了解一下一个好的工程应该有代码之外的东西

比如：
 - 使用文档
 - 测试用例
 - 自动化部署
 - 不同环境的配置的管理
 - 敏感信息的安全存储

另外，写完上面这些，其实可以帮助理清思路，代码的目的是什么，不能只限于无头绪的自娱自乐。
